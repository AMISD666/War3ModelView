## 4. 经验与建议 (Experience & Advice)

### 💡 核心库 `war3-model`
- **注意**: 这个库是编译后的 `.mjs` 文件。直接修改 `node_modules` 里的代码非常困难且容易被覆盖。
- **Patch 策略**: 如果必须修改库的行为，建议使用 `Proxy` 拦截或者像之前那样编写 `apply_patch.js` 脚本在 `postinstall` 时运行。但最好是向原作者提 Issue 或 PR。
- **渲染逻辑**: 它的渲染逻辑封装在 `ModelRenderer` 类中，很多属性是私有的，扩展时可能需要 `any` 类型强行访问。

### 💡 Rust 后端
- **Tauri 命令**: 所有涉及系统级操作（注册表、文件系统、控制台）都应放在 Rust 端实现，前端通过 `invoke` 调用。
- **依赖管理**: 修改 `src-tauri/Cargo.toml` 后，记得运行 `cargo check` 确保依赖正确（如 `windows-sys` 需要特定的 feature）。

### 💡 调试技巧
- **前端**: 使用 Chrome DevTools。
- **后端**: 使用我们添加的“显示调试控制台”功能，或者在终端看 `cargo run` 的输出。
- **热重载**: Vite 的 HMR 有时会失效（特别是修改 Rust 后），遇到奇怪问题先重启 `npm run tauri dev`。
