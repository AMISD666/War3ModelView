# War3ModelView 回归清单

每次涉及主编辑链路、节点编辑器、Viewer、批量模式或窗口通信的改动前后，都至少走完一次本清单。

## 使用方式

1. 复制“执行记录模板”到当前任务说明、PR 描述或 issue 评论。
2. 勾选实际执行的检查项，并在“结果”栏记录通过、失败或跳过原因。
3. 如果发现问题，补充复现步骤、影响范围和修复状态。

## 1. 模型导入与保存

- 导入一个 `MDX` 模型，确认主界面、节点树、Viewer 都正常加载。
- 导入一个 `MDL` 模型，确认不会出现白屏、卡死或主要面板缺失。
- 执行“保存模型”，确认原路径保存成功且模型可重新打开。
- 执行“另存为 MDL” 与 “另存为 MDX”，确认导出文件都能重新导入。
- 执行“一键 MDL/MDX 互转”，确认转换后的文件可再次打开。
- 打开“当前模型 TXT 文本”，确认文本视图可加载、关闭后主界面仍可继续操作。

## 2. 节点与粒子编辑

- 打开普通节点编辑对话框，修改名称、父节点、轴心点后保存，确认 Viewer 和节点树同步更新。
- 打开 `ParticleEmitter2` 编辑器，调整颜色、透明度、缩放、贴图等常用字段，确认实时预览正常。
- 在粒子编辑器中保存预设，确认不报错、弹窗可关闭。
- 打开关键帧编辑器并保存一次，确认返回主编辑器后数据仍可继续编辑。
- 独立节点窗口执行 `open -> edit -> close -> reopen`，确认数据没有丢失或异常重置。
- 独立节点窗口执行一次撤销/重做，确认不会影响主窗口可用性。

## 3. Viewer 与主界面

- 切换“查看模式 / 顶点模式 / UV 模式 / 动画模式 / 批量模式”，确认主区域都能正常渲染。
- 切换网格、节点、骨架、碰撞体、灯光、粒子、丝带、线框模式，确认按钮状态与 Viewer 表现一致。
- 选择一个动画并播放/暂停，确认 Viewer 没有卡死或明显异常闪烁。
- 在“查看模式”点击“左右镜像 / 垂直镜像”，确认模型、骨架动作和关键帧驱动的位移方向一起翻转，撤销/重做仍正常。
- 打开几何体可见性工具、模型变换、贴图动画管理器、动作管理器，确认单个弹窗异常不会拖垮整个应用。
- 在加载模型过程中观察 Loading 覆盖层，确认不会残留在界面上。

## 4. 材质、贴图与批量模式

- 打开贴图管理器、材质管理器，做一次简单修改后关闭，确认主模型仍可保存。
- 打开“全局颜色调整”，分别勾选/取消贴图、粒子、丝带、灯光等类型，拖动滑块后在鼠标松开时确认 Viewer 预览更新；执行一次保存后重新打开模型，确认颜色修改已真正写入。
- 执行“清理未使用的材质 / 贴图”，确认操作结果提示正常，模型没有被清空。
- 进入批量模式，滚动模型卡片列表并切换不同模型预览，确认没有明显卡顿或窗口冻结。
- 批量模式切换全视图，再切回普通视图，确认右侧预览仍正常工作。

## 5. 故障隔离验证

- 人工让某个高频面板抛出运行时错误时，确认只出现局部错误兜底，不应整页白屏。
- 主应用入口、独立工具窗口、主编辑区至少各验证一次 Error Boundary 能显示重试界面。
- Error Boundary 触发后，点击“重试”或刷新页面，确认应用能恢复到可继续操作的状态。

## 6. 发布前最低要求

- 运行一次构建或类型检查，至少保证当前分支可构建。
- 如果本次改动涉及文案，确认新增文案优先放在统一常量文件，不再散落写死。
- 如果本次改动涉及新弹窗、新面板或独立窗口，确认已评估是否需要接入 Error Boundary。
